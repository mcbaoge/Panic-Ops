# Panic Ops

一个给玩家改善高难整合包体验的 **Minecraft 联动辅助插件**（网页控制台 + WebSocket 实时控制）。

---

## 1. 这是做什么的？

`PanicOps` 提供一个本地网页控制面板，让玩家在手机或浏览器里更轻松地处理高压场景，例如：

- 清理/击杀附近生物（含敌对与全部生物）
- 查看玩家状态（血量、饥饿度、装备/手持耐久）并实时调整
- 手持物品附魔（支持超原版等级）
- 药水效果管理（清空、添加、调级）
- 传送到出生点与自定义传送点管理（新建/修改/删除/传送）
- 设备配对与个人设备管理（保障只由本人设备操作）

---

## 2. 主要功能

### 网页控制台

- 设备配对（配对码 / 二维码）
- WebSocket 实时连接状态
- 玩家状态实时面板
- 附近生物雷达热力图（红=敌对，黄=中立，绿=被动）
- 紧急操作：
  - 清除附近敌对生物
  - 击杀附近敌对生物
  - 清除附近所有生物
  - 击杀附近所有生物
- 实用工具：
  - 游戏模式切换（生存/创造/冒险/旁观）
  - 设置白天/夜晚
  - 清除天气
- 药水效果模块：
  - 外部显示已有效果并可 `+/-`
  - 弹窗添加未有效果（支持 Mod 药水）
  - 长按 `+/-` 支持步进 `10`
- 传送模块：
  - 一键回世界出生点
  - 自定义传送点：新建/修改/删除/传送

### 游戏内联动与保护

- 设备连接时聊天提示，可点击 `/kide <deviceId>` 立即踢出并删除设备
- `/panicops` 指令用于二维码、设备、待配对管理
- `/kide` 快速移除异常连接设备
- 支持命令 Tab 补全（含子命令、deviceId、待配对 code）

---

## 3. 设备配对流程（推荐）

1. 玩家进服后，聊天会收到配对码和二维码链接。  
2. 手机打开链接（或 `/panicops qr` 打开最新二维码页）。  
3. 页面输入配对码并完成绑定。  
4. 绑定后设备会被信任，并可直接进入控制台。  

---

## 4. 游戏内命令

### `/panicops`

```text
/panicops qr
/panicops device list
/panicops device kick <deviceId>
/panicops device remove <deviceId>
/panicops pending list
/panicops pending remove <code>
```

### `/kide`

```text
/kide <deviceId>
```

用途：立即踢出该设备并删除信任记录。

---

## 5. 传送点与权限隔离说明

- 传送点会持久化保存在配置文件中，重启后仍然可用。
- 传送点按设备归属玩家隔离：
  - 只能读取/修改/删除自己的传送点
  - 无法看到其他玩家的传送点

---

## 6. 关键配置文件

默认在游戏目录 `config/` 下：

- `panicops_devices.json`：已信任设备
- `panicops_teleports.json`：传送点数据

## 8. 常见问题

### Q1: 网页按钮点了没反应？

- 先确认网页显示已连接。
- 确认设备已完成配对并被信任。
- 修改后请重启服务端，确保资源与逻辑都加载最新版本。

### Q2: 发现异常设备连接怎么办？

- 直接点击聊天里的 `/kide <deviceId>` 超链接，或手动输入命令执行。

---

## 9. 适用场景

- 高难整合包中，玩家希望保留挑战但减少重复折磨
- 单人/小团体游玩时，用手机进行快速辅助操作
- 手机端快速处置危险生物/状态异常
- 无需进入控制台即可进行关键操作
